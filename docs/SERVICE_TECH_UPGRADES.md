# Service And Tech Upgrades

## 1. 升级目标

本轮升级把项目从“宠物服务原型”推进到“平台型产品骨架”。重点围绕四类使用者建立清晰的服务能力、数据模型和代码边界，让后端、MySQL 和网页端都能支撑后续大规模项目演进。

四类使用者：

- 普通用户：照护宠物、购买用品、查找医院、跟踪订单。
- 宠物用品店家：管理店铺、商品、库存、商城订单和售后。
- 宠物医院：管理医院资料、预约意向、资质和医疗服务内容。
- 管理员：审核供给侧、处理投诉、查看审计日志和运营数据。

## 2. 服务能力提升

### 2.1 普通用户

- 宠物档案从资料卡升级为平台核心资产。
- 支持疫苗、驱虫、绝育、过敏、病史和喂养偏好。
- 根据宠物类型和健康标签推荐商品、医院和上门服务。
- 订单中心聚合服务订单、商城订单和医院预约。
- 服务履约支持签到、照片回传、服务报告、评价和投诉。

### 2.2 店家

- 店家入驻和资质审核。
- 商品 SPU/SKU、库存、价格和上下架。
- 商城订单、发货、退款和售后。
- 经营指标：订单量、GMV、库存预警、售后率。
- 后续支持优惠券、推荐位和订阅购。

### 2.3 宠物医院

- 医院入驻和执业资质审核。
- 医生、科室、设备和营业时间管理。
- 预约意向处理和到院状态追踪。
- 疫苗、驱虫、体检、绝育等服务套餐。
- 医疗内容和健康提醒。

### 2.4 管理员

- 四类账号统一管理。
- 店家、医院、服务人员审核。
- 投诉、退款、风控、黑名单。
- 平台配置：分类、公告、活动、优惠券、推荐位。
- 数据看板：用户、订单、GMV、履约、售后、审核效率。
- 高风险操作写入审计日志。

## 3. 信任与履约机制

宠物服务的关键是用户是否敢把宠物、钥匙和健康信息交给平台生态。优先建议：

- 实名认证、资质认证、服务人员培训记录。
- 上门签到定位和服务轨迹。
- 服务过程照片或视频回传。
- 标准服务报告。
- 紧急联系人和异常上报。
- 平台保障说明、取消规则和仲裁流程。
- 店家商品资质、医院执业资质和评价反作弊。

## 4. 技术架构提升

### 4.1 后端

- 继续使用 FastAPI + MySQL。
- 保持 `domain`、`application`、`infrastructure`、`interfaces` 分层。
- 领域层只表达业务规则，不依赖 FastAPI、SQLAlchemy 或第三方 SDK。
- 应用层编排用例，依赖 repository port。
- 基础设施层实现 MySQL、Redis、对象存储、支付、地图、消息适配器。
- 接口层只处理 HTTP、鉴权、DTO 和异常映射。

### 4.2 数据库

- MySQL 8.0、InnoDB、utf8mb4。
- 金额使用整数分。
- 核心状态使用枚举字符串。
- 核心交易表保留审计字段。
- 初始化 SQL 仅用于本地演示，后续使用 Alembic 管理版本。

### 4.3 缓存与异步

下一阶段建议引入 Redis：

- 验证码。
- token 黑名单。
- 热门商品、热门医院和首页推荐缓存。
- 订单防重复提交。
- 库存和预约时段短锁。
- 限流。

建议引入 Celery 或 RQ：

- 订单超时关闭。
- 服务前提醒。
- 支付回调后续处理。
- 短信、App Push、小程序订阅消息。
- 图片压缩和服务报告归档。
- 数据统计。

### 4.4 前端

当前网页端是静态 MVP。正式项目建议拆分为：

- `web-user`：普通用户网页端。
- `merchant-web`：店家工作台。
- `hospital-web`：医院工作台。
- `admin-web`：平台管理后台。

统一技术栈：

- React + TypeScript + Vite。
- TanStack Query 管理服务端状态。
- Zustand 管理局部 UI 状态。
- Ant Design 用于管理后台。
- OpenAPI 生成 API 类型。

## 5. 本轮程序升级内容

- MySQL 初始化脚本包含四角色、权限、宠物、商品、医院、订单、预约、投诉和审计样例数据。
- 后端提供四角色登录注册接口。
- 后端提供角色工作台查询接口。
- 后端提供商品和医院目录查询接口。
- 网页端提供四角色登录注册、工作台展示、服务、商城、医院和订单概览。
- Demo 登录从默认自动降级改为显式勾选。
- 本地脚本支持重启、初始化数据库和端口清理。

## 6. 后续优先级

1. 接入 JWT access token + refresh token。
2. 建立路由级 RBAC 权限校验。
3. 引入 Alembic 管理 MySQL schema。
4. 补充认证、注册、工作台、目录接口测试。
5. 将静态网页端迁移为 React + TypeScript。
6. 接入 Redis 和异步任务。
7. 增加支付幂等、库存扣减和预约时段冲突测试。
