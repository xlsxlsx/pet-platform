# Pet Platform Task Spec

## 1. 里程碑

### M0 项目初始化

- 建立 `backend/`、`web/`、`docs/`、`scripts/` 目录。
- 建立 FastAPI 后端结构。
- 建立静态 Web MVP。
- 建立 MySQL 初始化脚本。
- 建立本地可见终端启动说明、停止说明和数据库初始化脚本。
- 输出 PRD、设计规范、技术栈、架构和开发规范文档。

### M1 四角色账号

- 支持普通用户、店家、医院、管理员四类角色。
- 支持手机号 + 网页图形验证码 + 短信验证码登录注册。
- 本地开发使用可调试短信验证码；生产环境替换为真实短信服务商并关闭验证码调试输出。
- 支持四角色注册。
- 注册时自动创建对应 profile。
- 管理员注册受配置开关保护。
- 根目录维护 `TEST_ACCOUNTS.md`。

### M2 四角色工作台

- 普通用户工作台：宠物、服务订单、商城订单、医院预约、提醒。
- 店家工作台：订单、库存、售后、商品表现。
- 医院工作台：预约、到院、资质、服务内容。
- 管理员工作台：审核、投诉、退款、GMV、审计。
- 后端提供 `GET /api/v1/dashboards/{role}`。
- Web 根据登录角色加载对应工作台。

### M3 目录能力

- 商品目录查询。
- 医院目录查询。
- Web 展示商品、医院、服务和订单概览。
- 数据来自 MySQL 种子数据。

### M4 上门服务 MVP

- 服务人员资料。
- 服务列表和筛选。
- 服务订单创建。
- 支付模拟。
- 服务报告。
- 评价和投诉。
- 管理员审核服务人员。

### M5 商城 MVP

- 商品 SPU/SKU。
- 购物车。
- 商城订单。
- 库存扣减。
- 发货、物流、确认收货。
- 退款和售后基础流程。

### M6 医院 MVP

- 医院资料管理。
- 医生和科室管理。
- 医院查询。
- 预约意向。
- 疫苗、驱虫、体检提醒。

### M7 多端完善

- React Web 用户端。
- 独立管理后台。
- App。
- 小程序。
- 多端登录状态和接口契约一致。

## 6. 验收标准

- 本地项目使用可见终端启动：先执行 `.\scripts\init_mysql.ps1 -User root -Password <your-local-mysql-password>`，再分别启动后端和网页服务。
- Web 可访问 `http://127.0.0.1:5173`。
- API 健康检查返回正常。
- `TEST_ACCOUNTS.md` 中四个角色账号均可登录。
- 登录注册均使用手机号验证码，不依赖密码字段。
- 注册接口可创建普通用户、店家和医院账号。
- 未开启配置时管理员注册返回 403。
- 四角色工作台均返回对应指标和任务。
- 商品和医院目录接口返回 MySQL 数据。
- 后端代码通过 `compileall`。
- 前端 `web/app.js` 通过 `node --check`。
- 文档无乱码、无过期或冲突的技术选型描述。
